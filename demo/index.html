What is Claude Code?

Anthropic's official CLI for agentic software engineering — runs in your terminal, reads and writes files, executes commands, and orchestrates specialized sub-agents.

Agents

Autonomous sub-processes with their own system prompts, tool access, and persistent memory. Spawned by an orchestrator or triggered by user intent.

Hooks

Shell scripts that fire on PreToolUse or PostToolUse events. Can block tool calls (exit 2) or run side-effects automatically.

Agent Memory

Markdown files that persist across conversations. Each agent owns a memory directory — institutional knowledge that survives session boundaries.

Skill Files

Custom slash commands (/feature, /review) that inject prompt templates. For repetitive user-triggered workflows.

Bounded Contexts

identity account category transaction budget reporting shared

Key Constraints

  • Hexagonal Architecture (ADR-014) — ArchUnit enforced
  • Domain zone = pure Java (zero Spring / JPA / Lombok)
  • Money = BigDecimal always — no float/double
  • Soft-delete only — is_active = false
  • All financial writes @Transactional
  • Auth via opaque UUID Bearer token (no JWT, no Spring Security)
03

The Challenge

Hexagonal Architecture with strict ArchUnit enforcement means dozens of rules to hold in mind while coding every feature.

{context}/
  domain/                 ←  PURE JAVA — zero Spring, JPA, Lombok, Jackson
  adapter/
    inbound/web/          ←  REST controllers + *RequestDto / *ResponseDto (records)
    outbound/
      persistence/        ←  *JpaEntity  +  *JpaMapper  +  *PersistenceAdapter
      event/              ←  Domain event publishers
      crosscontext/       ←  ACL adapters (no direct domain imports across contexts)
  config/                 ←  *Config.java — @Bean wiring ONLY (never @Service on domain)
✗ Without Claude Code
Spring annotations leak into domain · wrong layer for @Transactional · cross-context direct imports · float used for money · migration files edited in-place
✓ With Claude Code
7 specialized agents enforce rules at every phase · 3 pre-hooks block violations before they land · ArchUnit is the final safety net, not the first line of defence

  4. Domain service (zero Spring)

  8. REST controller + DTO records

  1. TypeScript types
  2. API client function
  3. Custom hook (use*.ts)

Automated Guardrails — Hooks

Shell scripts that fire automatically on tool events. exit 2 blocks the tool call and surfaces the reason to Claude.

PreToolUse Edit | Write pre-domain-purity.sh Spring/JPA/Lombok imports + @Service, @Entity, @Autowired in domain/*.java

PreToolUse Bash pre-bash-guard.sh Destructive git commands (reset --hard, push --force) + hard-delete SQL

Pre-hook: Domain Purity Guard

PreToolUse Edit | Write on */domain/*.java

Fires before Claude writes to any domain package Java file. Checks new_string (Edit) or content (Write) for forbidden patterns.

FORBIDDEN=(
  "import org.springframework"    # Spring beans / annotations
  "import javax.persistence"      # JPA (Jakarta EE)
  "import jakarta.persistence"    # JPA (Jakarta EE 9+)
  "import org.hibernate"          # Hibernate-specific
  "import lombok"                 # Lombok
  "import com.fasterxml.jackson"  # Jackson JSON
  "@Service"  "@Component"  "@Repository"  "@Autowired"
  "@Entity"   "@Table("     "@Column("
)

# If any match found in new content → exit 2 (block) + explanation to Claude
Example block message:
BLOCKED: Domain purity violation in BudgetDomainService.java
Forbidden: @Service
Domain classes must be pure Java. Wire via @Bean in BudgetConfig.java instead.
12

Pre-hook: Migration Immutability Guard

PreToolUse Edit | Write on */db.changelog/changes/*.yml

Liquibase computes a checksum for each changeset on first run. Modifying an existing migration breaks the checksum and causes every deployment to fail.

Blocked — file already exists

Edit: 005_create_accounts_table.yml

BLOCKED: Migration file is immutable:
005_create_accounts_table.yml

Create a new migration instead:
  changes/011_<description>.yml

Allowed — new file

Write: 011_add_budget_rollover_fields.yml
# File does not exist yet → allow

Migration sequence:
001  create_schema           ✓
002  create_user_tables      ✓
...
010  create_budgets_table    ✓
011  add_rollover_fields     ← NEW

The hook also suggests the next sequence number by scanning existing files in the changes/ directory.

13

Pre-hook: Destructive Command Guard

PreToolUse Bash

Blocks irreversible commands before they execute. Claude must ask for user confirmation instead of acting autonomously.

Blocked git operations

  • git reset --hard
  • git push --force / -f
  • git clean -f
  • git checkout -- .
  • git restore .
  • git branch -D
  • git rebase -i

Blocked SQL operations

  • DELETE FROM
  • DROP TABLE
  • TRUNCATE
This project uses soft-delete only (is_active = false). Hard deletes should never reach the finance_tracker schema.
Block message tells Claude: "Ask the user to confirm and run it directly as ! <command>"

MEMORY.md — Institutional Knowledge

Each agent embeds a ## MEMORY.md section at the bottom of its agent file. Agents update it as they discover stable patterns. Contents are loaded on every session start.

# Tech Lead Memory — Personal Finance Tracker
Last Updated: 2026-03-19

## Key Architecture Decisions
- BudgetApplicationService pattern: @Service @Transactional wrapper in config layer
  that delegates to BudgetCommandService (pure domain service, @Bean in BudgetConfig)
- BudgetPeriod.toMonthlyMultiplier(): pure BigDecimal enum. WEEKLY = 13/3. No float.
- percentUsed as double: acceptable for display-only computed percentages (not monetary)
- CategorySummary: lives in budget/domain/port/outbound/ — owned by the budget domain

## Migration Sequence (next = 011)
001 create_schema ✓   002 create_user_and_session_tables ✓   ...   010 create_budgets_table ✓

## Known Bugs (Phase 0)
BUG-001: application.yaml changelog path wrong (changelog/ vs db.changelog/)

Summary

What Claude Code provides

  • 7 specialized agents — each owns a phase of delivery
  • Mandatory 7-phase pipeline — no shortcuts, no phase skipping
  • 3 pre-hooks — block violations at edit time, faster than ArchUnit
  • 1 post-hook — auto-fix TypeScript on every edit
  • Persistent memory — agents accumulate project knowledge across sessions
  • Feature Briefs — versioned, user-confirmed spec before any code is written

.claude/agents/ ← 7 agent skill files .claude/hooks/ ← 4 hook scripts .claude/agent-memory/ ← persistent context .claude/settings.local.json ← hook wiring + permissions CLAUDE.md ← project rules (checked in)
